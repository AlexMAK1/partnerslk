
*Суть проблемы
Организация занимается мелкооптовой торговлей практически со всеми регионами России, ведя базу данных в 1С.
Прайс-лист представлен в формате xls, который рассылается клиентам еженедельно по электронной почте.
Получив свежий прайс, клиенты в строке с интересующим товаром и в отдельной колонке указывают необходимое
количество для заказа товара. После этого, в этом же файле, клиент кликает на кнопку (сам файл с макросами),
происходит формирование отдельного файла «лист заказа»,
который в свою очередь отправляется менеджеру для о последующего оформления в 1С документов.


Задача: уйти от формирования файла и рассылки прайс-лист для большинства клиентов, сделав для них личный кабинет.

Примерно описание функционала
Приложение имеет несколько сервисов (товары, цены, клиенты) для наполнения своей базы данных.
Клиент открывает личный кабинет (ЛК), создает новый заказ, указывает необходимое количество интересующего товара.
Подтверждает его.
Новый заказ забирает 1С.
Менеджер оформляет документы в 1С, при этом может отметить какие товары есть в наличии, а каких нет.
Отправляет для согласования клиенту.
Клиент подтверждает заказ в своем ЛК
После отгрузки товаров в 1С, клиент в ЛК видит pdf отгрузочных документов (счет, УПД)
